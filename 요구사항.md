# 디자인 패턴 PoC 어플리케이션 — 요구사항 정의서

> 최종 업데이트: 2026-04-09 | **V1 완료**

---

## 1. 프로젝트 개요

Java Spring Boot 프로젝트 개발 시 활용할 수 있는 디자인 패턴 학습 및 설계 지원 도구.  
PoC(Proof of Concept) 수준의 경량 웹 어플리케이션으로 개발하며, 3개의 핵심 기능을 단계적으로 구현한다.

---

## 2. 개발 단계 및 버전 계획

| 버전 | 포함 기능 | 상태 |
|------|-----------|------|
| **V1** | 기능 1 | ✅ 완료 |
| **V2** | 기능 1 + 기능 3 | 🔲 예정 |
| **V3** | 기능 1 + 기능 2 + 기능 3 | 🔲 예정 |

## 3. 기능 요구사항

### 3.1 기능 1 — 패턴 검색 및 예제 확인

#### 목적
GoF 23개 디자인 패턴에 대한 Java Spring Boot 예제 코드를 제공한다.

#### 요구사항

| ID | 요구사항 | 우선순위 | 상태 |
|----|----------|----------|------|
| F1-01 | GoF 23개 디자인 패턴을 카테고리(Creational / Structural / Behavioral)별로 목록으로 제공한다. | 필수 | ✅ |
| F1-02 | 패턴 목록에서 선택하거나, 검색창에서 검색하여 패턴 예제를 조회할 수 있다. | 필수 | ✅ |
| F1-03 | 검색창 입력 시 실시간 목록 필터링 및 Enter/버튼으로 즉시 로드가 모두 지원되어야 한다. | 필수 | ✅ |
| F1-04 | GoF 23개 패턴 예제는 사전 작성된 정적 데이터로 제공하며, AI API를 호출하지 않는다. | 필수 | ✅ |
| F1-05 | 모든 패턴 예제는 앱 기동 시점에 즉시 조회 가능하다 (별도 생성 대기 없음). | 필수 | ✅ |
| F1-06 | 예제 코드는 Java Spring Boot 기준으로 작성된다. | 필수 | ✅ |
| F1-07 | 예제 코드의 패키지 구조는 `contract / application / domain / infra / presentation` 5개 레이어를 기본으로 하되, 해당 패턴에 적합한 레이어만 사용한다. | 필수 | ✅ |
| F1-08 | 예제 코드 복사 기능을 제공한다. | 필수 | ✅ |
| F1-09 | 예제 코드를 전체화면으로 볼 수 있는 기능을 제공한다. 전체화면 내에서도 복사 가능하며 ESC 또는 닫기 버튼으로 종료한다. | 필수 | ✅ |
| F1-10 | 예제 코드는 max-height 제한 없이 전체 펼침으로 표시하며, 페이지 스크롤로 탐색한다. | 필수 | ✅ |
| F1-11 | Spring 전용 패턴은 현재 버전에서 제외한다. | 제외 | - |

#### GoF 23개 패턴 목록

| 카테고리 | 패턴 |
|----------|------|
| Creational (생성, 5개) | Singleton, Factory Method, Abstract Factory, Builder, Prototype |
| Structural (구조, 7개) | Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy |
| Behavioral (행동, 11개) | Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor |

#### 예제 조회 결과 화면 구성 (순서)

| 순서 | 섹션 | 내용 |
|------|------|------|
| 01 | 패턴 개요 | 패턴의 의도 및 정의 (한국어) |
| 02 | Spring Boot 활용 시나리오 | Spring Boot에서의 사용 시점과 이유 (한국어) |
| 03 | 패키지 구조 | 디렉토리 트리 형태 (ASCII) |
| 04 | 핵심 이점 | 패턴 적용의 주요 장점 (한국어) |
| 05 | 예제 코드 | Java Spring Boot 전체 코드 + 복사 버튼 + **전체화면 버튼** |

#### 예제 코드 전체화면 기능

| 항목 | 설명 |
|------|------|
| 진입 | 예제 코드 섹션 우상단 `전체화면` 버튼 클릭 |
| 표시 | 다크 배경 풀스크린 오버레이, 패턴명 + 언어 배지 표시 |
| 기능 | 코드 스크롤, 복사 버튼 |
| 닫기 | `✕ 닫기` 버튼 또는 `ESC` 키 |

### 3.2 기능 2 — 분석자료 기반 패키지 구조 시각화

#### 목적
업로드된 기능 분석 자료를 AI가 분석하여 적합한 디자인 패턴을 추천하고,  
선택된 패턴 기반의 Java Spring Boot 패키지 구조를 시각화한다.

#### 요구사항

| ID | 요구사항 | 우선순위 | 상태 |
|----|----------|----------|------|
| F2-01 | CSV 또는 Excel 형식의 분석 파일을 업로드할 수 있다. | 필수 | 🔲 |
| F2-02 | 업로드된 파일에서 `호출구분 / 구분1(고객유형) / 구분2(서비스상세) / 구분3(단말유형) / 구분4(기타) / L4` 컬럼을 Factor로 분석한다. | 필수 | 🔲 |
| F2-03 | Anthropic Claude API를 활용하여 분석 자료 기반으로 적합한 디자인 패턴을 추천한다. | 필수 | 🔲 |
| F2-04 | 추천된 패턴 목록에서 사용자가 하나의 패턴을 선택한다. | 필수 | 🔲 |
| F2-05 | 선택된 패턴과 분석 자료를 결합하여 Java Spring Boot 패키지 구조를 생성한다. | 필수 | 🔲 |
| F2-06 | 패키지 구조를 트리 형태와 다이어그램(박스+선) 두 가지 방식으로 시각화한다. | 필수 | 🔲 |
| F2-07 | 생성된 패키지 구조에 대한 검토 의견, 장단점, 고려사항을 분석하여 표시한다. | 필수 | 🔲 |
| F2-08 | 패키지 구조 시각화는 UI에서만 제공하며, 실제 파일/폴더를 생성하지 않는다. | 필수 | 🔲 |

#### 분석 파일 스펙

- **지원 형식**: CSV, Excel (.xlsx)
- **인코딩**: UTF-8 (BOM 포함 가능)
- **기준 파일**: `functionAnaylyzeCheck.csv`, `functionAnaylyzeProvision.csv`, `functionAnaylyzeSave.csv`
- **주요 컬럼**:

| 컬럼명 | 역할 |
|--------|------|
| AS-IS PGM | 프로그램 ID |
| AS-IS PGM명 | 프로그램 명칭 |
| Depth | 호출 깊이 |
| AS-IS PGM 함수 | 함수 ID |
| AS-IS PGM 함수명 | 함수 명칭 |
| 설명 | 기능 설명 |
| 호출구분 (PR/CO) | **Factor** — PR: 접수, CO: 개통 |
| 구분1 (고객유형) | **Factor** |
| 구분2 (서비스상세) | **Factor** |
| 구분3 (단말유형) | **Factor** |
| 구분4 (기타) | **Factor** |
| L4 | **Factor** — L4 코드 |
| L4의 기능 | L4 설명 |

### 3.3 기능 3 — 분석정리 MD 생성

#### 목적
기능 1과 기능 2의 결과를 마크다운 형식으로 다운로드한다.

#### 요구사항

| ID | 요구사항 | 우선순위 | 상태 |
|----|----------|----------|------|
| F3-01 | 기능 1에서 조회한 패턴 예제를 마크다운 파일로 다운로드할 수 있다. | 필수 | 🔲 |
| F3-02 | 기능 2에서 생성한 패키지 구조 시각화 및 분석 의견을 마크다운 파일로 다운로드할 수 있다. | 필수 | 🔲 |
| F3-03 | 기능 1과 기능 2의 다운로드는 각각 별도로 제공한다. | 필수 | 🔲 |
| F3-04 | 여러 항목 중 원하는 항목만 선택하여 다운로드할 수 있다. | 필수 | 🔲 |
| F3-05 | 선택 저장 예시: A패턴 검색결과 저장 / B패턴 검색결과 저장 / 파일 분석 후 패키지 구조 저장 — 각각 독립적으로 선택·다운로드 가능 | 필수 | 🔲 |

## 4. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **경량성** | PoC 검증 목적으로 최대한 가벼운 기술 스택 사용 |
| **즉시성** | 패턴 예제는 사전 탑재 정적 데이터로 제공하여 대기 없이 즉시 조회 |
| **무비용** | 기능 1 예제 조회 시 AI API를 호출하지 않아 API 비용 미발생 |
| **확장성** | 규칙 기반(Rule-based) 패턴 추천으로 향후 변경 가능한 구조 (기능 2) |

---

## 5. 제약 사항

- Java Spring Boot 기준의 예제 코드만 제공 (타 언어/프레임워크 미지원)
- GoF 23개 패턴만 지원 (Spring 전용 패턴 제외)
- 패키지 구조 시각화는 UI 표현에 한정 (실제 코드/파일 생성 없음)
- Anthropic API Key는 `.env` 파일로 관리 (기능 2에서만 사용)
